# PL/EN — Plan wersji językowych

**Cel:** aplikacja ma działać po polsku i angielsku, bez mieszania języków w UI i bez psucia promptów do AI.

## Zakres proponowany

- [ ] Dodać ustawienie języka interfejsu: `pl` / `en`.
- [ ] Przenieść teksty UI do mechanizmu tłumaczeń Django (`gettext`, pliki `.po`).
- [ ] Dodać przełącznik języka w nagłówku.
- [ ] Zdecydować osobno język promptów AI:
  - UI może być PL/EN,
  - prompt do Gemini/OpenAI powinien domyślnie używać języka procesu/czynności albo ustawienia operacji, żeby model zwracał spójne nazwy czynności.
- [ ] Dodać test smoke: strona listy procesów i formularz czynności renderują kluczowe etykiety po polsku i angielsku.

## Uwagi

- Najpierw warto domknąć stabilność przepływu analizy i korekty.
- Nie tłumaczyć nazw procesów, operacji i czynności wpisanych przez użytkownika automatycznie; to dane domenowe.
- Przy eksporcie CSV można dodać język nagłówków jako opcję później.
